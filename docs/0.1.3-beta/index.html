Getting Started

An overview of Rex CSS Framework, how to download and use, basic templates and examples, and more.

This is the first BETA release of ReX Css Framework.

The purpose of the Rex CSS Framework Beta project is to be a reference and get feedback on the quality and usability from developers. In order to establish a useful and flexible solution for Css styles, so please be aware of the following indications before installing it:

  • This is an experimental project, please be aware of that if you plan to use it in production.
  • This project provides a basement of a real life Css library, please feel free to give us any feedback.
  • Use it to evaluate the code structure, check distribution method (NPM), dependencies and implementation process in your project.
  • The code structure, variables and any code related content can be changed in the next iterations in order to evaluate features, bugs fixes or improvements.
  • This is the primary version of the Css library (Core element, Components, Helpers), the full version will be created based on your feedback.
  • We tested the styles in multiple and different real devices, browsers and by using BrowserStack, even do some code might not work as expected, so if you find some bugs please create an issue.

We appreciate your feedback!

How it was built

Rex Css Framework -BETA- is based on Bootstrap v3 and we added our customizations on top of the framework.
The main project uses Sass files to manage the variables, dependencies and different output files such as rex.css (Full version) or rex-typography.css (Typography styles only). For fast implementantion use the distribution CSS files from the CDN or 'dist/' folder from NPM.
The project only provides Static CSS Styles.
For any JavaScript interaction we are working on React Components based on the next STABLE version of the framework that will be based on Bootstrap v4 using UI components instead of a Monolithic CSS Framework (current BETA version).

This is the most basic form of Rex CSS Framework.

How to layout grids

  • The container class gives its div a specific fixed width. The container-fluid's width changes every time the viewport width changes.
  • The row class is used mainly to hold columns in it.
  • Grid columns are created by specifying the number of 12 available columns you wish to span. For example, three equal columns would use three .col-sm-4
